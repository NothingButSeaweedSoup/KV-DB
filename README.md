# KV-DB：一个高性能的键值存储数据库

仲恺农业工程学院  计算机科学与技术2023级Java高级编程课程作业

KV-DB 是一个基于 LSM 树的高性能键值存储数据库，支持单机模式和集群模式。它提供了丰富的功能，包括数据持久化、查询优化、数据一致性保障以及易用的 API，适用于多种应用场景。

## 功能特点

### 单机模式

- **数据库基本功能**：
    - 支持键值对的增、删、改、查操作。
- **数据持久化**：
    - 数据自动写入磁盘，支持文件自动 Rotate 和压缩。
    - 使用多线程技术对 Rotate 文件进行压缩，提高性能。
- **查询效率优化**：
    - 设计了高效的索引机制，通过 SSTable 和布隆过滤器加速查询。
    - 提供数据缓存，优先从缓存中查找数据。
- **数据一致性**：
    - 使用 WAL日志实现数据一致性，确保异常情况下数据不丢失。
    - 支持异常恢复，保障内存与磁盘数据一致性。
- **存储结构优化**：
    - 支持多种数据类型（如字符串、列表、映射等）。
    - 采用 LSM 树结构，优化读写性能。
- **易用的 API**：
    - 提供 CLI 命令行工具，方便用户操作。
    - 提供 Shell 脚本 `dbcli.sh`，快速启动 CLI 工具。
    - 提供 Java 语言的 Jar 包，方便集成到其他项目中。
    - 提供基于 Socket 的网络 API，支持远程访问。

### 集群模式

- **角色设计**：
    - 支持主节点和从节点，主节点负责读写操作，从节点作为备份。
- **角色选定**：
    - 通过配置文件静态配置主从节点，简单易用。

## 快速开始

### 单机模式

1. **编译项目**：
   ```bash
   mvn clean install
   ```
2. **启动 CLI 工具**：
   ```bash
   ./dbcli.sh
   ```
3. **使用 CLI 工具操作数据库**：
    - 插入键值对：`put <key> <value>`
    - 查询键值：`get <key>`
    - 删除键值：`delete <key>`
    - 刷新数据到磁盘：`flush`
    - 退出工具：`exit`

### 集群模式

1. **配置集群**：
    - 编辑 `cluster-config.json` 文件，配置主节点和从节点信息。
2. **启动集群**：
   ```bash
   java -cp target/classes server.DBServer
   ```
    - 根据配置文件启动主节点和从节点。
3. **使用客户端连接集群**：
    - 启动客户端工具，连接到主节点进行操作。

## 技术栈

- **Java**：核心开发语言。
- **LSM 树**：存储引擎采用 LSM 树结构。
- **WAL 日志**：使用 Write-Ahead Logging 保障数据一致性。
- **布隆过滤器**：加速查询效率。
- **多线程**：用于文件压缩等操作。
- **Socket**：网络通信。
